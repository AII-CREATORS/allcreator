# Weekly Commit Summary (주간 커밋 요약)

지난 기간의 git 커밋을 수집해서, **날짜별로 "그날 수행한 기능"을 핵심 키워드로 요약**한
한국어 리포트를 생성한다. 파일 목록이 아니라 **무슨 일을 했는지**가 한눈에 보이게 한다.

## 언제 쓰나

"지난 일주일 뭐 했는지 정리해줘", "이번 주 작업 요약", "최근 커밋 정리" 같은
회고·리뷰·릴리스노트 성격의 요청. 기본 기간은 **7일**이지만 사용자가 "지난 3일",
"이번 달", "6월 커밋"처럼 다르게 말하면 그 기간을 따른다.

## 절차

### 1. 파라미터 파악
- **기간**: 기본 `7 days ago`. 사용자가 다른 기간을 말하면 `--since`/`--until`로 반영.
- **브랜치/작성자**: 언급되면 각각 브랜치 이름, `--author`로 필터. 언급 없으면 현재 브랜치 전체.
- 먼저 저장소가 맞는지, 커밋이 있는지 확인한다. 커밋이 하나도 없으면 그 사실을 그대로 알린다.

### 2. 데이터 수집
커밋별 날짜·작성자·제목·본문을 뽑는다. 요약은 **커밋 메시지 중심**이므로 파일 목록은
기본적으로 필요 없다.

```bash
git log --since="7 days ago" --date=short \
  --pretty=format:'%x1f%ad%x1f%an%x1f%s%x1f%b%x1e'
```

- 기간을 바꿀 땐 `--since` 값만 교체한다. 종료일이 필요하면 `--until="YYYY-MM-DD"` 추가.
- 커밋 메시지가 부실해 무슨 기능인지 애매하면, 그 커밋만 `git show --stat <hash>`로
  변경 파일을 봐서 성격을 보강한다.
- 병합(merge) 커밋의 노이즈를 줄이려면 필요 시 `--no-merges`를 붙인다. 기본은 포함.

윈도우 PowerShell 환경에서도 git은 동일하게 동작하지만, `%x1f` 구분자 파싱이
번거로우면 Bash 도구로 실행하는 편이 안정적이다. 정 어렵다면
`git log --since=... --date=short --pretty=format:'%ad %s'`로 날짜+제목만 먼저 봐도 된다.

### 3. 날짜별로 그룹화
커밋을 **날짜(YYYY-MM-DD)로 묶는다.** 같은 날 여러 커밋이 있으면 한 날짜 그룹 아래
모아서 정리한다. 날짜는 **최신순**으로 배치한다.

### 4. 기능 키워드 요약
각 날짜 아래에, 그날 커밋들이 수행한 **기능**을 짧은 불릿으로 적는다.

- **핵심 키워드로만** 쓴다. 문장으로 길게 풀지 말고 명사구 위주로 짧게.
  (예: "댓글 기능 구현", "이메일 인증 도입", "카테고리 다중선택")
- 커밋 메시지를 그대로 복붙하지 말고, 여러 커밋의 비슷한 작업은 **하나로 합쳐** 정리한다.
- **커밋 해시는 붙이지 않는다.** 파일 경로도 싣지 않는다 — 기능만 남긴다.
- 한 날짜의 기능이 많아도 대표 항목 위주로 3~6개 이내로 압축한다.

## 출력 형식

```markdown
# 주간 작업 요약 (2026-07-08 ~ 2026-07-14)

**대상**: 브랜치 `dev` · 커밋 7개 · 작성자 1명

## ✅ 2026-07-14
- 프롬프트 삭제 기능 정리
- 카테고리 다중선택
- 댓글 입력창 UI 수정

## ✅ 2026-07-13
- 댓글 기능 구현
- 답글·알림 설정
- 페이지네이션 리로드

## ✅ 2026-07-10
- 이메일 인증 도입
- 트래픽 이슈 대응
- 보안 정비(RLS·RPC)

## 🔎 한눈에 보기
- 이번 주 핵심: 댓글 시스템 구축 + 이메일 인증 전환
```

- 날짜 그룹은 **최신순**, 각 날짜 헤더 앞에 초록 체크 `✅`를 붙인다.
- 기능 불릿은 **핵심 키워드**로만, 짧게.
- 마지막 "한눈에 보기"에서 그 기간의 흐름을 1~2줄로 짚어 회고에 바로 쓰이게 한다.
