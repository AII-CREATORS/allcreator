# WBS Commit (주차별 진척 요약)

기준 커밋을 시작점으로, **거기서부터 최신 커밋까지를 주(week) 단위로 쪼개어**
각 주차에 무슨 기능·세팅을 했는지 **한 줄씩** 요약한다. 진척율과 일정 관리(WBS)를
위한 리포트이므로, 세세한 나열보다 "그 주에 큰 틀에서 무엇을 했나"가 보이게 한다.

## 언제 쓰나

"이 해시부터 지금까지 주차별로 정리해줘", "커밋 기준 진척 요약", "WBS로 뽑아줘" 등
시작 커밋을 기준으로 일정·진행 상황을 파악하려는 요청. `commit-week`가 날짜별 상세
요약이라면, 이 스킬은 **주차 단위의 굵직한 진척 요약**이다.

## 절차

### 1. 기준 커밋·파라미터 파악
- **기준 해시**: 사용자가 준 커밋 해시가 시작점. 없으면 물어본다.
- **브랜치/작성자**: 언급되면 반영. 언급 없으면 현재 브랜치 기준.
- 먼저 그 해시가 저장소에 존재하는지 확인한다. 없으면 그 사실을 그대로 알린다.

```bash
git rev-parse --verify <hash>^{commit}   # 존재 확인
```

### 2. 기간 계산
기준 커밋의 날짜와 최신 커밋(HEAD)의 날짜를 구해, 그 사이를 대상으로 삼는다.

```bash
git show -s --format=%ad --date=short <hash>   # 기준 커밋 날짜
git show -s --format=%ad --date=short HEAD      # 최신 커밋 날짜
```

### 3. 커밋 수집
기준 커밋(포함)부터 HEAD까지의 커밋을 **오래된 순**으로 날짜·제목·본문과 함께 뽑는다.
(`<hash>^..HEAD`로 기준 커밋 자체도 포함시킨다. 기준이 최초 커밋이면 `^`를 빼고 `<hash>..HEAD`에
기준을 별도로 더한다.)

```bash
git log --reverse --date=short \
  --pretty=format:'%x1f%ad%x1f%an%x1f%s%x1f%b%x1e' \
  <hash>^..HEAD
```

- 커밋 메시지가 부실하면 그 커밋만 `git show --stat <hash>`로 변경 파일을 봐서 보강.
- 병합 커밋 노이즈가 심하면 `--no-merges`를 붙인다.

### 4. 주차로 그룹화
수집한 커밋을 **주(week) 단위로 묶는다.**

- **1주차 = 기준 커밋 날짜부터 7일**, 2주차 = 그다음 7일 … 식으로 기준일 기준 7일 간격으로 나눈다.
  (달력상 월요일 시작이 아니라 **기준일 기준 롤링 7일**로 계산해 진척을 직관적으로 본다.)
- 각 주차의 실제 날짜 범위(YYYY-MM-DD ~ YYYY-MM-DD)를 함께 표기한다.
- 커밋이 없는 주차는 "작업 없음"으로 남겨 공백도 일정상 드러나게 한다.

### 5. 주제별 요약 (주차 h1 · 주제 h2 · 내용 h3)
각 주차를 **주제(기능/세팅) 단위로 쪼갠다.** "그 주 전체를 한 줄로" 뭉치지 말고,
초기세팅·로그인·결제 … 처럼 그 주에 다룬 작업을 주제별로 나눈다.

- **주차는 h1**(`# 📌 N주차 (기간)`), **각 주제는 h2**(`## 주제명`),
  **그 주제의 내용은 h3**(`### 한 줄 설명`)로 낸다.
- 비슷한 커밋은 같은 주제로 합치고, 주제가 다르면 h2를 새로 만든다.
- 세부 커밋 나열·해시·파일 경로는 넣지 않는다. 한 주제는 한 줄로 압축.

## 출력 형식

주차 h1 → 주제 h2 → 내용 h3의 순수 헤딩 구조로 낸다.

```markdown
# WBS 진척 요약 (기준 `5742c93` · 2026-06-29 ~ 2026-07-14)

**대상**: 브랜치 `dev` · 총 3주차 · 커밋 51개

# 📌 1주차 (06-29 ~ 07-05)

## 초기 세팅
### Supabase 연동, 전역 CSS·toast 등록

## 로그인
### 이메일·구글·카카오 API 구현

## 결제
### TOSS Payments 연동, 중복 구매 방지

# 📌 2주차 (07-06 ~ 07-12)

## 인증
### OAuth/세션/비번재설정 정비, 이메일 인증 도입

## 관리자
### 승인/거절 confirm 흐름

## 🔎 진척 메모
- 진행 흐름: 1주차에 로그인·프롬프트·결제까지 핵심 골격을 몰아서 구축 →
  2주차에 인증/세션을 대대적으로 손보며 안정화 →
  3주차에 댓글 같은 커뮤니티 기능으로 확장. 초반에 기능이 집중되고
  후반으로 갈수록 안정화·확장 성격으로 무게중심이 이동함.
- 공백 주차: 없음 (3주 연속 작업)
```

- 주차는 **오래된 순(1주차부터)**, 각 주차는 h1 `# 📌 N주차 (기간)`.
- 주차 안은 **주제 h2 + 내용 h3**. 커밋이 없는 주차는 "(작업 없음)"으로 표기.
- **진척 메모의 "진행 흐름"은 한 줄로 끝내지 말고 구체적으로 쓴다.** 주차 간 성격 변화
  (기능 집중 → 안정화 → 확장 등)와 그렇게 판단한 근거·부가설명을 2~4줄로 풀어,
  일정·진척 판단에 바로 쓰이게 한다. 공백 주차나 특이점도 함께 짚는다.
